§ tile_copy_fn

typedef void(* tile_copy_fn) (uint32_t x0, uint32_t x1, uint32_t x2, uint32_t x3, uint32_t y0, uint32_t y1, char *dst, const char *src, int32_t linear_pitch, uint32_t swizzle_bit, mem_copy_fn mem_copy)

Each row from y0 to y1 is copied in three parts: [x0,x1), [x1,x2), [x2,x3).

These ranges are in bytes, i.e. pixels * bytes-per-pixel. The first and last ranges must be shorter than a "span" (the longest linear stretch within a tile) and the middle must equal a whole number of spans. Ranges may be empty. The region copied must land entirely within one tile. 'dst' is the start of the tile and 'src' is the corresponding address to copy from, though copying begins at (x0, y0). To enable swizzling 'swizzle_bit' must be 1<<6, otherwise zero. Swizzling flips bit 6 in the copy destination offset, when certain other bits are set in it.

§ intel_get_memcpy()

bool intel_get_memcpy ( mesa_format  tiledFormat,

Determine which copy function to use for the given format combination.

The only two possible copy functions which are ever returned are a direct memcpy and a RGBA <-> BGRA copy function. Since RGBA -> BGRA and BGRA -> RGBA are exactly the same operation (and memcpy is obviously symmetric), it doesn't matter whether the copy is from the tiled image to the untiled or vice versa. The copy function required is the same in either case so this function can be used.

Parameters
[in]tiledFormatThe format of the tiled image
[in]formatThe GL format of the client data
[in]typeThe GL type of the client data
[out]mem_copyWill be set to one of either the standard library's memcpy or a different copy function that performs an RGBA to BGRA conversion
[out]cppNumber of bytes per channel
Returns
true if the format and type combination are valid

Copy from linear to tiled texture.

Divide the region given by X range [xt1, xt2) and Y range [yt1, yt2) into pieces that do not cross tile boundaries and copy each piece with a tile copy function (tile_copy_fn). The X range is in bytes, i.e. pixels * bytes-per-pixel. The Y range is in pixels (i.e. unitless). 'dst' is the start of the texture and 'src' is the corresponding address to copy from, though copying begins at (xt1, yt1).

Copy texture data from linear to X tile layout.

Each row from y0 to y1 is copied in three parts: [x0,x1), [x1,x2), [x2,x3). These ranges are in bytes, i.e. pixels * bytes-per-pixel. The first and last ranges must be shorter than a "span" (the longest linear stretch within a tile) and the middle must equal a whole number of spans. Ranges may be empty. The region copied must land entirely within one tile. 'dst' is the start of the tile and 'src' is the corresponding address to copy from, though copying begins at (x0, y0). To enable swizzling 'swizzle_bit' must be 1<<6, otherwise zero. Swizzling flips bit 6 in the copy destination offset, when certain other bits are set in it.

The mem_copy parameters allow the user to specify an alternative mem_copy function that, for instance, may do RGBA -> BGRA swizzling. The first function must handle any memory alignment while the second function must only handle 16-byte alignment in whichever side (source or destination) is tiled.
