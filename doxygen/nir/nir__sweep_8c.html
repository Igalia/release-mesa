Detailed Description

The nir_sweep() pass performs a mark and sweep pass over a nir_shader's associated memory - anything still connected to the program will be kept, and any dead memory we dropped on the floor will be freed.

The expectation is that drivers should call this when finished compiling the shader (after any optimization, lowering, and so on). However, it's also fine to call it earlier, and even many times, trading CPU cycles for memory savings.
