Default ctx->Driver.AllocTextureStorage() handler.

The driver can override this with a more specific implementation if it desires, but this can be used to get the texture images allocated using the usual texture image handling code. The immutability of GL_ARB_texture_storage texture layouts is handled by texObj->Immutable checks at glTexImage* time.

GLsizei  height,
GLsizei  depth 
)
inlinestatic

Texture width, height and depth check shared with the multisample variants of TexStorage functions.

From OpenGL 4.5 Core spec, page 260 (section 8.19) 

"An INVALID_VALUE error is generated if width, height, depth
or levels are less than 1, for commands with the corresponding
parameters."

(referring to TextureStorage* commands, these also match values specified for OpenGL ES 3.1.)
