This pass adds <0.5, 0.5> to all uses of gl_FragCoord.

Run before nir_lower_io().

For a more full featured pass, consider using nir_lower_wpos_ytransform(), which can handle pixel center integer / half integer, and origin lower left / upper left transformations.

This simple pass is primarily intended for use by Vulkan drivers on hardware which provides an integer pixel center. Vulkan mandates that the pixel center must be half-integer, and also that the coordinate system's origin must be upper left. This means that there's no need for a uniform - we can always just add a constant. In the case that sample shading is enabled, Vulkan expects FragCoord to include sample positions.
