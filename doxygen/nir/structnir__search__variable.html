Optional condition fxn ptr.

This is only allowed in search expressions, and allows additional constraints to be placed on the match. Typically used for 'is_constant' variables to require, for example, power-of-two in order for the search to match.

§ type

nir_alu_type nir_search_variable::type

Indicates that the given variable must have a certain type.

This is only allowed in search expressions and indicates that the given variable is only allowed to match values that come from an ALU instruction with the given output type. A type of nir_type_void means it can match any type.

Note: A variable that is both constant and has a non-void type will never match anything.

Referenced by match_value().
