Detailed Description

Represents a double-ended queue of unique blocks.

The worklist datastructure guarantees that eacy block is in the queue at most once. Pushing a block onto either end of the queue is a no-op if the block is already in the queue. In order for this to work, the caller must ensure that the blocks are properly indexed.
