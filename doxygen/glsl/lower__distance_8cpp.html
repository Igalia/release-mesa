Detailed Description

This pass accounts for the difference between the way gl_ClipDistance is declared in standard GLSL (as an array of floats), and the way it is frequently implemented in hardware (as a pair of vec4s, with four clip distances packed into each).

The declaration of gl_ClipDistance is replaced with a declaration of gl_ClipDistanceMESA, and any references to gl_ClipDistance are translated to refer to gl_ClipDistanceMESA with the appropriate swizzling of array indices. For instance:

gl_ClipDistance[i]

is translated into:

gl_ClipDistanceMESA[i>>2][i&3]

Since some hardware may not internally represent gl_ClipDistance as a pair of vec4's, this lowering pass is optional. To enable it, set the LowerCombinedClipCullDistance flag in gl_shader_compiler_options to true.
