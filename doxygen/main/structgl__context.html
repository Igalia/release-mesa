Detailed Description

Mesa rendering context.

This is the central context data structure for Mesa. Almost all OpenGL state is contained in this structure. Think of this as a base class from which device drivers will derive sub classes.

§ _Shader

struct gl_pipeline_object* gl_context::_Shader

Current active shader pipeline state.

Almost all internal users want ::_Shader instead of ::Shader. The exceptions are bits of legacy GLSL API that do not know about separate shader objects.

If a program is active via glUseProgram, this will point to ::Shader.

If a program pipeline is active via glBindProgramPipeline, this will point to ::Pipeline.Current.

If neither a program nor a program pipeline is active, this will point to ::Pipeline.Default. This ensures that ::_Shader will never be NULL.
